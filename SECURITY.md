# Security Policy

## Supported versions

The latest published release branch is the only supported version for security fixes.

## Reporting a vulnerability

Please do not open a public issue for a suspected security problem.

Instead:

1. Prepare a minimal description of the issue.
2. Include affected files, inputs, and reproduction steps if available.
3. Contact the maintainer through the private channel listed on the repository profile once the public GitHub repository is created.

Until the repository owner information is finalized, treat security reporting details as pending release setup work.
