# Security Policy

## Supported versions

The latest published release branch is the only supported version for security fixes.

## Reporting a vulnerability

Please do not open a public issue for a suspected security problem.

Instead:

1. Prepare a minimal description of the issue.
2. Include affected files, inputs, and reproduction steps if available.
3. Report the issue privately by email to `linzaixinroyleo@foxmail.com` with the subject line `[VisionLabelOps security]`.

I will review reports as quickly as practical and follow up with next-step guidance or a fix timeline. For non-security bugs and feature requests, please use the public GitHub issue tracker.
